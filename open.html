How to Contribute — Quick Guide

1. Find an Issue
Look for labels: good first issue, help wanted or beginner.
2. Fork & Clone
Fork the repo, clone locally, and create a topic branch for your work.
3. Run Tests
Run the project tests or linters. Add tests for your changes when possible.
4. Make PR
Open a pull request with a clear title and description referencing the issue number.

Need a template? Click to generate a starter ISSUE_TEMPLATE.md for your repo.

Open Source License

Prefer permissive licenses (MIT, Apache 2.0) for small projects. Include a LICENSE file and contributor covenant if you expect community involvement.

Maintainer Tips

  • Label issues clearly and add contribution docs.
  • Keep a CONTRIBUTING.md and CODE_OF_CONDUCT.md.
  • Use CI to validate PRs and friendly templates to speed reviews.
